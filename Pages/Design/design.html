Font & Color Choices

I'm really into the idea of crafting a super modern website. I'm all about that flow where everything just clicks together – from the vibrant colors playing off each other to the fonts that perfectly complement the vibe. Right now, I'm in that sweet spot of exploration, seeing what works and what doesn't. But I gotta say, I'm liking where I'm at with it all.

Style

Minimalism? Oh yeah, that's totally my jam. I'm all about keeping things clean and sleek, stripping away the excess to focus on what really matters. But here's the thing – I don't want to sacrifice excitement for simplicity. Finding that sweet spot where minimalism meets creativity is my goal. And you know what? I'm all for exploring different avenues, not just sticking to one medium. Mixing things up keeps it fresh and adds layers to my academic work.

Approach

I feel like a good approach to this all is to create color palettes to keep track of it all. So in Blog Post Week 3, you can check it out! I show a lot more of my process here.

Code Break Down

HTML

Key Points:

  • Metadata: The *head* section contains all my metadata, including my viewport settings, information about the author, and a character set.
  • Styling: I have in place the framework for my CSS filing.
  • Script: I have my script.js where it imports the Javascript module and initializes it with the function initialise (then the page it is on).
  • Structure: The page has a nav menu, a dynamically generated menu container, the main content section, and then my footer.

Javascript

Key Points:

  • Root Path: The root constant stores the base path of the website, which is used as a prefix for all page URLs.
  • Pages Array: The pages array contains objects representing different pages with their names and relative URLs.
  • Navigation Initialization Function: The initialise function dynamically generates a navigation menu based on the provided page data.
